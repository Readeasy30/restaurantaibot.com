# RestaurantAIBot Restaurant Profile Template

Last updated: 2026-05-28

## Purpose

This file defines the safe future structure for restaurant profile pages.

Restaurant profile pages are planned marketing assets. They are not live paid listings unless the page clearly says so, has been manually approved, and follows disclosure rules.

## Required profile sections

Each future restaurant profile should include:

1. Restaurant name
2. City and neighborhood
3. Cuisine or food category
4. Short plain-language description
5. Good-for tags such as lunch, dinner, takeout, family meals, coffee, date night, or travel food
6. Menu highlights or signature items
7. Contact links such as website, phone, directions, order link, or reservation link
8. Owner-approved offer text if any
9. Clear label if the profile is sponsored, paid, or promoted
10. Confirm-before-you-go warning

## Safe wording rules

Use clear wording:

- Demo Profile
- Restaurant Profile
- Featured Restaurant only when approved
- Sponsored only when paid and disclosed
- Confirm details directly with the restaurant

Avoid unsupported claims:

- best restaurant
- guaranteed open
- cheapest
- safest
- allergy-safe
- official listing unless verified

## Future build order

1. Create reusable profile layout.
2. Create manually reviewed profile data.
3. Add profile pages only after owner approval.
4. Add sponsored labels when money or other compensation is involved.
5. Add payment only after legal, privacy, disclosure, and business rules are ready.

## Blocked until approval

Do not add:

- live paid listings
- payment checkout
- hidden sponsored placement
- campaign tracking scripts
- advertiser dashboard
- unapproved restaurant claims
- private API keys
