# Restaurant customer site template

Use this folder as the build standard for restaurant websites sold through RestaurantAIBot.

## Template purpose

Create fast, mobile-first restaurant websites using plain HTML, CSS, and JavaScript.

## Standard pages

- Home
- Menu
- Catering and private events
- Gallery
- Contact and directions
- Privacy
- Terms

## Standard files

- `index.html`
- `menu.html`
- `catering.html`
- `gallery.html`
- `contact.html`
- `privacy.html`
- `terms.html`
- `css/style.css`
- `robots.txt`
- `sitemap.xml`
- `llms.txt`
- `README.md`
- `STATUS.md`

## Build rules

- Use only facts supplied by the restaurant owner.
- Do not invent reviews, awards, photos, prices, hours, menu items, delivery promises, allergy claims, or health claims.
- Keep the site fast and mobile-first.
- Keep language simple.
- Use clear call, directions, order, and reservation buttons.
- Link to existing ordering and reservation tools instead of building ordering from scratch.
- Use owner-provided photos only.

## Starter homepage structure

1. Hero with restaurant name, cuisine, city, short promise, call button, directions button.
2. Quick facts: cuisine, neighborhood, hours, best for.
3. Menu highlights.
4. Current offer if supplied.
5. Ordering and reservation links if supplied.
6. Location and contact.
7. Short FAQ.
8. Disclaimer that details can change.

## SEO standard

Every customer site needs:

- One clear H1.
- Plain answer near top.
- Local page title.
- Meta description.
- Internal links.
- FAQ section.
- Sitemap.
- robots.txt.
- llms.txt.

## Customer build checklist

Before building:

- Intake is complete.
- Restaurant facts are clear.
- Photos are owner-provided or placeholders are used.
- Domain decision is clear.
- Customer has approved the service level.

After building:

- Check all nav links.
- Check mobile layout.
- Check call button.
- Check map/directions link.
- Check ordering/reservation links.
- Update STATUS.md.
